Ruby's #map Method

November 21, 2015

Imagine you have an array, and you'd like to do something to the entire array. Well, the ruby Array method #map makes it very easy for you to do so! What #map will do is take an array as well as a block and return to you a new array that will contain values that are affected by the block you pass. Here is an example:

> numbers_array = [1,2,3,4,5]
> numbers_array.map{|each_number| each_number * 2}

output:
=> [2, 4, 6, 8, 10]

As you can see, numbers_array is an array on which #map is called. In return, we get an array where each_number enumerated through *2, resulting in => [2, 4, 6, 8, 10] . It's as simple as that! This method is especially useful becuase it leaves the original array completely untouched. If you wanted to modify the original array, you can just use #map! instead. Think of the #map method in the same way as an #each method, but the #each method returns nothing while the collect method returns an array.

Something you may see often is the #collect method, which is used in the same exact way as the #map method. Keep this in mind, as they are just prefrences but essentially the two methods work in the same exact way. Happy Coding!
